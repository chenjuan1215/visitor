关于本项目

在传统的多页应用中，每个 URL 对应一个物理 HTML 文件。但在单页应用（SPA）中，整个应用只有一个 HTML 文件（index.html）。

当用户直接访问 /about 时，浏览器向服务器请求该路径。由于服务器上不存在 about.html，GitHub Pages 默认会返回 404 错误。

解决方案： 我们将 index.html 复制一份命名为 404.html。当服务器找不到文件时，会返回 404.html，而它的内容就是 SPA 的入口，从而让前端 JavaScript 有机会读取 URL 并渲染正确的组件。
